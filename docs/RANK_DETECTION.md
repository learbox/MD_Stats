# 段位图标检测技术说明

## 概述

独立线程持续截图，通过 OpenCV 模板匹配识别 Master Duel 对局中双方头像旁的段位图标和数字等级。

## 源素材

从 AssetRipper 解包获得 Unity RGBA 源素材（290×290，32 位带透明通道），存放在 `resource/templates/rankicons/`。

支持 8 种段位：新手、青铜、白银、黄金、铂金、钻石、大师、巅峰。

## 核心原理

### 1. RGBA 背景自适应合成

游戏 UI 背景色变化（主题切换等）不影响匹配。每帧从截图采样实际背景色，将透明通道素材合成到该背景上：

```
composite = source_bgr × alpha + bg_sample × (1 - alpha)
```

### 2. 两级搜索（金字塔加速）

- **粗搜**：截图缩放到 600px 宽，在缩略图上以步长 12 搜全部 8 种图标
- **精搜**：粗搜候选图标在候选尺寸 ±10 范围以步长 2 精确匹配
- **修正**：坐标映射回原图后用单模板修正偏移

总耗时约 0.5 秒（缓存命中），首次约 0.6 秒。

### 3. 位置缓存

首次检测到段位后自动记录 `(分辨率, 侧) → (x, y, 尺寸)` 到 `rank_positions.toml`。后续同分辨率检测仅搜索缓存位置附近 2 倍图标大小区域，跳过缩略图全搜。

### 4. 数字等级识别（列投影峰值法）

段位图标下方裁 22%×11% 区域，二值化后统计每列暗像素和（列投影），数峰值个数：

- 1 峰 = I
- 2 窄峰 = II
- 3 峰 = III
- 2 峰（一窄一宽）= IV

## 架构

```
RankDetector (QThread)
  ├─ 独立截图循环 (0.5s)
  ├─ 窗口最小化/关闭时暂停
  ├─ 检测到一方后跳过该侧继续搜另一方
  └─ 双方齐或阶段2踩停 → emit rank_icon_detected
         ↓
  main_window._on_rank_icon_detected()
         ↓
  状态栏: "段位: 铂金 II vs 钻石 I"
  CSV: 己方段位列 / 对方段位列
```

## 配置

```toml
[rank_detection]
enabled = true              # 启用段位检测
interval = 0.5              # 截图间隔（秒）
confidence_threshold = 0.7  # 匹配阈值
```

## 故障恢复

- 源素材缺失：静默跳过，不影响主流程
- 位置缓存被删：首次检测时自动重新生成
- 未检测到：CSV 段位列留空，不报错
