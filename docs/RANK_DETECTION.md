# 段位图标检测技术说明

## 概述

独立线程持续截图，通过 OpenCV 模板匹配识别 Master Duel 对局中双方头像旁的段位图标和数字等级。

## 源素材

从 AssetRipper 解包获得 Unity RGBA 源素材（290×290，32 位带透明通道），存放在 `resource/templates/rankicons/`。

支持 8 种段位：新手、青铜、白银、黄金、铂金、钻石、大师、巅峰。

> **注意**：因 KONAMI 版权，段位图标源素材不包含在仓库中。使用前需用 AssetRipper 从游戏资源包提取 9 张 PNG，放入 `resource/templates/rankicons/`。详见 `resource/templates/README.md`。

## 核心原理

### 1. RGBA 背景自适应合成

游戏 UI 背景色变化（主题切换等）不影响匹配。每帧从截图采样实际背景色，将透明通道素材合成到该背景上：

```
composite = source_bgr × alpha + bg_sample × (1 - alpha)
```

### 2. 两级搜索（金字塔加速）

- **粗搜**：截图缩放到 600px 宽，在缩略图上以步长 12 搜全部 8 种图标
- **精搜**：粗搜候选图标在候选尺寸 ±10 范围以步长 2 精确匹配
- **修正**：坐标映射回原图后用单模板修正偏移

总耗时约 0.5 秒（缓存命中），首次约 0.6 秒。

### 3. 位置缓存

首次检测到段位后自动记录 `(分辨率, 侧) → (x, y, 尺寸)` 到 `rank_positions.toml`。后续同分辨率检测仅搜索缓存位置附近 2 倍图标大小区域，跳过缩略图全搜。

### 4. 数字等级识别（列投影峰值法 + 峰宽分析）

段位图标下方裁取 22%×11.5% 区域（约 30~50 列 × 20~30 行）。

**步骤一：二值化**

```
OTSU 自动阈值 → 数字（白 255）+ 背景（黑 0）
```

**步骤二：列投影**

把每一列的黑色像素数加起来，得到一条一维曲线。有笔画的地方投影值高（峰），没笔画的地方投影值低（谷）：

```
   II 的二值化图          列投影曲线
   ┌──────┐              ↑
   │ █  █ │              │   ██      ██
   │ █  █ │              │   ██      ██     ← 2 个峰 = II
   │ █  █ │              │  ████    ████
   └──────┘              └──────────────────→ 列
```

**步骤三：数峰**

遍历投影曲线，超过最大值 35% 的连续隆起算一个峰，跌到 50% 以下算离开。得到峰数 n。

**步骤四：量峰宽**

每个峰的宽度（列数）= 该隆起占了多少列。

**步骤五：联合判定**

直观对比——每个等级的二值化图和列投影：

```
  I:  █              ██              → 1 窄峰
      
  V: ╱ ╲          ████████           → 1 宽峰（平顶山）
     ╲╱                                    

  II: █ █          ██    ██          → 2 等宽峰

  IV: █╱╲         ██  ██████         → 2 峰,宽窄比>1.8

  III:█ █ █       ██  ██  ██         → 3 峰
```

```
n_peaks  峰宽特征                   → 等级  (实测例子)
───────  ────────────────────────   ─────  ──────────────────
  1     峰宽 < 字符区 25%            I      竖线投影窄，占10~20%
  1     峰宽 ≥ 字符区 25%            V      两斜线融合为平顶山,占30~50%
  2     两峰等宽,宽窄比 ≤ 1.8        II     两根分离竖线
  2     宽峰/窄峰 > 1.8              IV     I竖线(窄) + V形(宽)
  3                                  III    三根分离竖线
```

**为什么 V 只有 1 个峰？**

```
   V 形正面      俯视（每列投影）
    ╱ ╲           ╱██████╲
   ╱   ╲    →    ╱████████╲     两斜线从顶到底在
  ╱     ╲       ╱██████████╲    每一列都重叠 → 只产
 ╱       ╲                          生 1 个宽峰
```

V 的两条斜线不是垂直线——从顶部展开到底部相交。中间部分的每一列被两条斜线同时覆盖，形成一片连续的高原（"平顶山"），不分叉。

**为什么 II 和 IV 峰数相同但能区分？**

```
  II:  ∣  ∣     → 两个都窄 (~13px)，比值 ≈ 1.0
  IV:  ∣  V     → 一个窄(~12px) + 一个宽(~28px)，比值 ≈ 2.3
```

IV 的 V 形投影宽度远大于 I 竖线。当 `宽峰/窄峰 > 1.8` 时判定为 IV，否则为 II。

**置信度计算**

```
置信度 = (0.5 + 0.5 × 离判定边界的距离) × 峰谷清晰度

例子：
  宽窄比=2.3 → 离1.8边界=0.5 → 基础分值0.75 × 清晰度0.9 = 0.68 ✓
  宽窄比=1.82→ 离1.8边界=0.02→ 基础分值0.51 × 清晰度0.7 = 0.36 ✗ (丢弃)
```

置信度 < `confidence_threshold` 时丢弃等级(tier=None)，CSV 只写段位名不带数字。

## 架构

```
RankDetector (QThread)
  ├─ 独立截图循环 (0.5s)
  ├─ 窗口最小化/关闭时暂停
  ├─ 检测到一方后跳过该侧继续搜另一方
  ├─ 双方齐 → emit rank_icon_detected + 暂停
  └─ 对局结束 → resume_for_next_game() 清缓存恢复
         ↓
  main_window._on_rank_icon_detected()
         ↓
  状态栏（默认）: "段位: 铂金 II vs 钻石 I"
  状态栏（show_confidence）: "段位: 铂金 (0.92) II (0.82) vs 钻石 (0.83) I (0.75)"
  CSV: 己方段位列 / 对方段位列
```

## 配置

```toml
[rank_detection]
enabled = true              # 启用段位检测
interval = 0.5              # 截图间隔（秒）
confidence_threshold = 0.7  # 图标 NCC 匹配阈值 + 等级判读阈值（共用）

[debug]
show_confidence = false     # 状态栏显示置信度分数
```

## 故障恢复

- 源素材缺失：静默跳过，不影响主流程
- 位置缓存被删：首次检测时自动重新生成
- 未检测到：CSV 段位列留空，不报错
